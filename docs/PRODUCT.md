# Ayuvam — Product Overview

*A clean client deliverables portal for studios, agencies, and freelancers.*

---

## The line

> **Make the work look as good as it is.**

Pronounced *Ah-yoo-vam*.

---

## What it does

Each client gets their own workspace. You drop proposals, decks, files, and links into folders. Your client opens one link — sees a clean, read-only view of everything. No login. No Drive folder chaos.

Three pillars. Only three.

| | What it means |
|---|---|
| **One workspace per client** | Isolated. Acme sees Acme's work, not Northwind's. |
| **Organized content** | Folders (infinite nesting). Items are links or files. Tagged, dated, searchable, deduplicated. |
| **Beautiful client access** | A magic link. No signup. Read-only. Revocable. |

---

## Who it's for

- Solo freelancers with 3–8 ongoing clients
- Design studios (2–10 people)
- Marketing or branding consultancies
- Agencies with retainer clients
- Founders who want to look more professional to their clients

---

## How a user gets started

```
1. Landing page → "Start free"
2. Continue with Google → consent screen
3. Onboarding → name first workspace
4. Land inside the workspace
5. Create folders → drop in proposals, decks, files
6. Click "Share with client" → enter their email
7. Client receives branded invite → clicks → sees their workspace
```

Total time from sign-up to first shared workspace: **~60 seconds.**

## Features

### Core
- Workspaces per client, isolated by design
- Folders, infinitely nested, color-coded
- Items: links (any URL) or files (up to 20 MB)
- Tags, notes, custom dates per item
- Pin important items to the top
- Full-text search (⌘K, eventually) across titles, tags, notes, URLs
- Duplicate-link detection — flags if a URL already exists, offers to add a note

### Sharing
- Generate share links per workspace
- Optional email invite via Resend (branded)
- Per-share label and tracked last-access time
- Revoke any link instantly
- Public client view is read-only by enforcement, not just UI

### Roles
- **Admin** — system-level (you, the platform owner)
- **Manager** — owns a workspace; can edit everything
- **Reader** — view-only collaborator (for team members)
- **Client** — no account; access via magic link only

### Team
- Invite teammates by email
- Each member is a manager or reader, scoped per workspace

### Brand-aligned details
- Branded HTML emails sent from `hello@ayuvam.com`
- "Powered by Ayuvam" badge on client view (removable on Agency plan)
- Optional custom subdomain on Agency plan

## How a typical day looks

```
You wrap a deliverable
   ↓
Drop the file or link in the client's workspace
   ↓
(First time only) "Share with client" → enter email
   ↓
Client gets the email → opens the link → sees only their work
   ↓
Next time, no email needed — client bookmarks the link
```

The product is daily-use for you, weekly-use for your client. The client never sees Ayuvam admin UI — only the clean read-only view.
